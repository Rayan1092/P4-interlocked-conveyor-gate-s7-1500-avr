# P4 Interlocked Conveyor Gate S7-1500 AVR

A two-station interlocked conveyor gate system built for a Siemens internship project. A Siemens S7-1500 PLC acts as the supervisory controller and an Arduino running bare-metal AVR C++ handles servo motion and ultrasonic sensing as a subordinate device. The two communicate over a custom 3-wire STROBE/DATA/ACK handshake protocol.

I built this without a Functional Design Specification. The full system design, architecture and code were worked out directly without any prior documentation.

Unfortunately I forgot to record a video of the system running connected to the PLC, so there is no demo footage.

## What it does

A servo-driven arm acts as a physical gate for a small conveyor belt. The PLC owns the conveyor motor and only allows it to run once the gate arm has moved to the clear position and the ultrasonic sensor confirms it. The Arduino handles all servo motion and sensor timing. The PLC handles all logic, interlocking, fault detection and the HMI.

The system runs a startup self-test sweep before allowing normal operation. Every fault gets logged with a fault ID, which system triggered it and a timestamp into a 20-slot circular buffer data block on the PLC.

## Physical setup

- 3D printed conveyor frame with a toothed belt driven by a DC motor
- Servo-driven gate arm that sweeps between home and clear positions
- Home position detected by a limit switch wired directly to the PLC
- Clear position detected by an ultrasonic sensor handled by the Arduino
- DC motor switched on/off through a relay, no speed control
- Arduino on a breadboard with voltage divider circuits to bring 24V PLC signals down to 5V logic and a MOSFET level shifter to bring the Arduino ACK output up to 24V for the PLC digital input

## PLC side (S7-1500, SCL)

- Reads limit switch and ultrasonic confirmation from the Arduino
- Runs a startup self-test sweep triggered from the HMI
- Sends position commands to the Arduino over a STROBE/DATA/ACK handshake. STROBE tells the Arduino to read, DATA carries the target position, ACK confirms receipt. The PLC tracks the ACK timeout with a TON timer and faults out if no response within 100ms
- Only enables the conveyor motor relay once the clear position is verified and no active fault exists
- Logs faults with fault ID, source system and timestamp into a 20-slot circular buffer array data block using a custom UDT
- Two-screen WinCC Unified HMI with live sensor status, servo position, conveyor state and controls for Start, Stop, Self-Test and Manual jog

## Arduino side (bare-metal AVR C++, avr-gcc)

No Arduino framework or libraries used anywhere.

- Custom millis() built from scratch using Timer0 in CTC mode with a compare-match ISR. No delay-based blocking
- Servo PWM generated by direct Timer1 register manipulation (WGM, COM, ICR1, OCR1A). No servo library
- Implements the STROBE/DATA/ACK protocol as the receiving end with rising-edge detection on STROBE
- Handles ultrasonic trigger pulse and echo measurement, reports a simple threshold yes/no back to the PLC

## Hardware

- PLC: Siemens SIMATIC S7-1500
- HMI: Siemens WinCC Unified
- Microcontroller: Arduino Uno (ATmega328P), programmed with avr-gcc bare-metal C++
- Conveyor: 3D printed frame, DC motor, toothed belt
- Gate: servo motor with 3D printed arm
- Sensors: limit switch (home), ultrasonic HC-SR04 (clear position)
- Interface: voltage dividers for 24V to 5V, MOSFET level shifter for 5V to 24V
